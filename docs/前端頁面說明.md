## 1. 頁面清單與功能說明

目前共有 **7 個頁面**，其中 2 個已正常啟用，2 個啟用但功能開發中，3 個尚未對外開放。

### 1.1 LoginPage（登入頁）
**路徑**：`/login`  
**狀態**：已啟用

這是整個系統的入口。未登入的使用者會被自動導到這個頁面。頁面提供 email / password 兩個欄位，送出後向後端驗證身份，成功後把回傳的 JWT token 存到 localStorage，然後跳到儀表板。

---

### 1.2 DashboardPage（儀表板）
**路徑**：`/`（根路徑）  
**狀態**：已啟用，主力頁面

整個系統最核心的頁面，分為兩大區塊：

**今日訊號區**
- 顯示今日評分前 3 檔的股票卡片（ScoreCard），標示「推薦買進」。
- 可展開查看「全部評分清單」，顯示所有股票的分數與訊號（買進 / 觀望 / 賣出）。
- 每張卡片有「確認買入」按鈕，分數達到買進門檻時才啟用。

**目前持倉區**
- 顯示目前持有的每一檔股票（PositionCard），包含成本價、現價、未實現損益。
- 每張卡片有「確認賣出」按鈕。

**其他操作**
- 右上角「重新整理」按鈕：重新呼叫 API 拉取最新訊號與持倉資料。
- 買入 / 賣出都會跳出確認視窗（Modal），確認後才真正送出交易。

---

### 1.3 SettingsPage（設定頁）
**路徑**：`/settings`  
**狀態**：已啟用

系統管理與控制中心，包含以下幾個功能區：

**監控股票清單管理**
- 顯示目前加入監控的股票列表（存在 localStorage）。
- 可以手動新增或刪除監控股票。

**策略參數設定**
- 設定買進門檻分數、停損比例、獲利目標比例。
- 點擊「儲存設定」後存到 localStorage。

**股票池篩選**
- 點擊「開始篩選」，讓後端重新篩選符合條件的股票池。
- 篩選過程約需數分鐘，前端會每隔 30 秒輪詢後端確認狀態，完成後重新載入股票池清單。
- 篩選進行中的狀態會以 `isFiltering` 旗標存在 localStorage，確保重新整理後不會遺失進度。

**全量評分掃描**
- 點擊「立即計算今日評分」，讓後端對所有股票跑完整的評分計算（約 5–10 分鐘）。
- 和股票篩選一樣，前端會每隔 30 秒輪詢確認完成，逾時（12 分鐘）後自動解除鎖定。
- 掃描開始時間戳存在 localStorage（`fullScanRunning`），重新整理後可繼續等待。

**系統資訊**
- 顯示 API 伺服器位址等基本資訊。

**登出**
- 清除 localStorage 的 token，跳回登入頁。

### API 通用說明

- 所有 API 請求都會自動帶上 `Authorization: Bearer <token>` 標頭（由 Axios 攔截器處理）。
- 如果後端回傳 `401 Unauthorized`，系統會自動清除 localStorage 的 token，並強制導回 `/login`。
- API 基底網址來自環境變數 `VITE_API_BASE_URL`（設定在 `.env` 檔案中）。
- 請求逾時設定為 **60 秒**。

### 3.2 認證流程（最重要的跳轉邏輯）

```
使用者開啟任何受保護頁面
        ↓
PrivateRoute 檢查 localStorage 是否有 token
        ↓
  有 token → 正常顯示頁面
  沒有 token → 強制導向 /login
        ↓
輸入 email / password → POST /api/v1/auth/login
        ↓
  成功 → 存 token → navigate("/")（儀表板）
  失敗 → 顯示錯誤訊息，留在登入頁
```

**登出流程：**
1. 在 SettingsPage 點擊「登出」
2. 刪除 `localStorage.token`
3. `navigate("/login")`

**Token 過期自動登出：**
1. 任何 API 呼叫收到後端回傳 `401`
2. Axios 攔截器刪除 `localStorage.token`
3. `window.location.href = "/login"`（強制整頁跳轉）

## 4. localStorage 儲存的資料與用途

目前系統在瀏覽器的 localStorage 中存了以下資料：

---

### 4.1 `token`

| 項目 | 內容 |
|------|------|
| **資料型態** | 字串（JWT token） |
| **用途** | 身份驗證，每次呼叫 API 時帶在 Header 裡 |
| **何時寫入** | 登入成功後（LoginPage） |
| **何時刪除** | 登出時、後端回傳 401 時 |
| **誰會讀取** | PrivateRoute（判斷是否已登入）、Axios 攔截器（帶入請求 Header） |

---

### 4.2 `watchList`

| 項目 | 內容 |
|------|------|
| **資料型態** | JSON 陣列，例如 `["2330", "2317", "0050"]` |
| **用途** | 儲存使用者自訂的監控股票清單（股票代碼列表） |
| **何時寫入** | 在 SettingsPage 新增或刪除股票時更新 |
| **何時刪除** | 不會主動刪除，使用者刪光後會是空陣列 |
| **誰會讀取** | SettingsPage 載入時、`src/utils/watchList.js` 工具函式 |
| **備註** | 支援舊格式轉換（若舊資料是物件格式，會自動轉成代碼字串陣列） |

---

### 4.3 `strategySettings`

| 項目 | 內容 |
|------|------|
| **資料型態** | JSON 物件，例如 `{ "buyThreshold": 70, "stopLoss": 5, "profitTarget": 10 }` |
| **用途** | 儲存策略參數：買進門檻分數、停損比例、獲利目標比例 |
| **何時寫入** | SettingsPage 點擊「儲存設定」時 |
| **何時刪除** | 不會主動刪除 |
| **誰會讀取** | SettingsPage 載入時、`src/utils/strategy.js` 工具函式（會與預設值合併，避免缺少欄位） |

---

### 4.4 `isFiltering`

| 項目 | 內容 |
|------|------|
| **資料型態** | 字串 `"true"` |
| **用途** | 標記「股票池篩選進行中」，讓頁面重新整理後知道篩選還沒結束 |
| **何時寫入** | SettingsPage 點擊「開始篩選」時 |
| **何時刪除** | 後端回報篩選完成時、或輪詢超時後 |
| **誰會讀取** | SettingsPage 載入時（若此旗標存在，會自動繼續輪詢篩選狀態） |

---

### 4.5 `fullScanRunning`

| 項目 | 內容 |
|------|------|
| **資料型態** | 字串（毫秒時間戳，例如 `"1714201200000"`） |
| **用途** | 記錄全量評分掃描的啟動時間，頁面重新整理後可繼續等待並計算是否已超時 |
| **何時寫入** | SettingsPage 點擊「立即計算今日評分」時，寫入當下的 `Date.now()` |
| **何時刪除** | 後端回報評分完成時、或啟動超過 12 分鐘（逾時自動清除） |
| **誰會讀取** | SettingsPage 載入時（若此值存在，會自動繼續輪詢評分狀態，並檢查是否已超時） |

---

### localStorage 完整彙整表

| 鍵名 | 類型 | 寫入時機 | 刪除時機 | 主要用途 |
|------|------|---------|---------|---------|
| `token` | 字串 | 登入成功 | 登出 / 401 | 身份驗證 |
| `watchList` | JSON 陣列 | 新增/刪除監控股票 | 不主動刪除 | 監控清單持久化 |
| `strategySettings` | JSON 物件 | 點擊儲存設定 | 不主動刪除 | 策略參數持久化 |
| `isFiltering` | 字串 "true" | 開始篩選 | 篩選完成/超時 | 篩選進行中狀態 |
| `fullScanRunning` | 時間戳字串 | 開始評分掃描 | 掃描完成/超時 | 評分掃描進行中狀態 |
